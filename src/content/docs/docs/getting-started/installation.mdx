本指南帮助你把 BangNiCMS 部署到一台普通云服务器上，**全程 10 分钟，无需写一行代码**。

## 你需要准备

- **服务器**：至少 2 核 4G 内存 / 50G 硬盘 / Ubuntu 22.04（任何主流云厂商皆可）
- **域名**：一台已备案的域名（如 `cms.example.com`），并能修改 DNS A 记录
- **SSH 工具**：Mac/Linux 自带 `ssh`；Windows 用户推荐 [Tabby](https://tabby.sh/) 或 PowerShell

## 完整部署步骤

<Steps>

1. **登录服务器，安装 Docker**

   首次使用云服务器时，先用 SSH 连进去：

   ```bash
   ssh root@你的服务器IP
   ```

   然后**一行命令**安装 Docker（官方脚本）：

   ```bash
   curl -fsSL https://get.docker.com | sh
   apt install -y docker-compose-plugin
   ```

   安装完后用 `docker --version` 确认能正常输出版本号。

2. **创建部署目录**

   把 BangNiCMS 的所有数据放在 `/opt/bangnicms/` 下，便于后续备份：

   ```bash
   mkdir -p /opt/bangnicms
   cd /opt/bangnicms
   ```

3. **下载 docker-compose.yml**

   ```bash
   wget https://cms.bonnei.com/docker-compose.yml
   wget https://cms.bonnei.com/.env.example -O .env
   ```

4. **生成并填入 3 个密钥**

   用以下命令一次性生成 3 个随机密钥：

   ```bash
   echo "BANGNICMS_POSTGRES_PASSWORD=$(openssl rand -hex 24)" >> .env
   echo "BANGNICMS_JWT_SECRET=$(openssl rand -hex 24)" >> .env
   echo "BANGNICMS_REVALIDATE_SECRET=$(openssl rand -hex 24)" >> .env
   ```

   然后用 `nano .env` 把里面的 `DOMAIN` 改成你的真实域名：

   ```diff
   - DOMAIN=cms.bonnei.com
   + DOMAIN=cms.example.com
   - ADMIN_EMAIL=admin@bonnei.com
   + ADMIN_EMAIL=admin@example.com
   ```

   保存后退出（Ctrl+X → Y → 回车）。

   <Aside type="caution">
   `.env` 文件包含敏感密钥，**永远不要提交到任何 git 仓库**。如果要备份服务器，单独加密保存。
   </Aside>

5. **启动**

   ```bash
   docker compose up -d
   ```

   第一次会拉 5 个镜像（postgres / redis / server / web / admin / caddy），大约 3-5 分钟。完成后用以下命令验证：

   ```bash
   docker compose ps
   ```

   你应该看到 6 个容器全部 `Up (healthy)`：

   ```
   NAME                    STATUS
   bangnicms-postgres      Up (healthy)
   bangnicms-redis         Up (healthy)
   bangnicms-server        Up (healthy)
   bangnicms-web           Up
   bangnicms-admin         Up
   bangnicms-caddy         Up
   ```

6. **配置 DNS**

   到你的域名注册商控制台（如阿里云 / DNSPod），添加两条 A 记录：

   | 主机记录 | 类型 | 值 | TTL |
   |---|---|---|---|
   | `cms`（或你的子域） | A | 服务器公网 IP | 600 |
   | `admin.cms` | A | 服务器公网 IP | 600 |

   等待 5-10 分钟 DNS 生效。

7. **首次访问 + 初始化**

   浏览器打开 `https://你的域名/install`（如 `https://cms.example.com/install`）。

   首次访问 Caddy 会自动从 Let's Encrypt 申请证书，可能需要等待 30-60 秒。

   按引导填入：站点名称、默认语言、管理员邮箱与密码。

   完成后跳转到后台首页，部署成功。

</Steps>

## 验证一切正常

打开几个关键端点，确认服务全部健康：

<Tabs>
  <TabItem label="健康检查">
    ```bash
    curl https://你的域名/health
    # 期望输出：{"status":"ok","uptime":...}
    ```
  </TabItem>
  <TabItem label="前台">
    在浏览器访问 `https://你的域名/`，应该看到默认主题的首页。
  </TabItem>
  <TabItem label="后台">
    访问 `https://admin.你的域名/`，用刚才的管理员邮箱登录。
  </TabItem>
</Tabs>

## 卸载与清理

<Aside type="danger">
卸载会**永久删除**所有数据（文章、产品、媒体库、用户等）。务必先备份。
</Aside>

```bash
cd /opt/bangnicms

# 1. 停止所有容器
docker compose down

# 2. 删除所有数据（如要彻底清理）
rm -rf data/

# 3. 释放镜像空间（可选）
docker image prune -a
```
